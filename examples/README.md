# Worked Examples

Worked examples live under `examples/` in the source repo. They are reference
material only and are not copied into installed target repositories.

Read them in this order:

1. [happy-path/README.md](./happy-path/README.md)
2. [fix-forward/README.md](./fix-forward/README.md)

Use the happy-path example first.
It shows one complete change that moves from proposal through rollup without a
rejected checkpoint or follow-up milestone.

Use the fix-forward example second.
It shows how the same artifact family behaves when execution lands only
partially, review requires fix-forward work, and rollup carries unresolved
items forward explicitly.
