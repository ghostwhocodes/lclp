# Technical Decision Log: <system or subsystem name>

> Artifact type: `durable technical context artifact`

## Purpose

This log records durable technical-design decisions and the reasoning behind
them.

Use it to capture decisions about:

- system boundaries
- invariants
- migration stance
- compatibility stance
- major structural tradeoffs
- interface patterns
- schema and contract rules
- naming conventions
- subsystem-specific technical rules
- system-overview transitions when versioning is used

Do not use it for:

- milestone planning
- temporary run notes
- product positioning decisions
